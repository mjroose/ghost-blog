Flatiron School Ruby CLI Project - KetoRecipe

I'm studying full stack web development in the Flatiron School's community-powered code bootcamp.  The first project in the course is a command line interface (CLI) application built in Ruby using object-oriented programming principles.  For this project, I decided to build an application that scrapes Category and Recipe information from Keto Connect, a blog that shares recipes featured on the Keto Connect YouTube channel.  The source code for this project is available at my GitHub repository.

How to Use the Application

To run the program, simply navigate to the project folder cloned from GitHub and type "ruby bin/recipe" in the command line.  The application will display a list of categories scraped, using Nokogiri, from the Keto Connect website.

After entering the category's number, the application displays a list of recipes in the selected category.

From there, simply enter the recipe's number, and the recipe will be printed to the screen.

How I Built the Application

As I built the application, I decided to create three classes:  Category, KetoRecipe, and Menu.  A Category has a name, URL, and many recipes.  I created a Category class method to scrape all of the recipe categories from Keto Connect.  Each Category knows what recipes are in the category using the :recipes attribute.

An instance of the KetoRecipe class has a name, URL, category, and lists of ingredients and instructions.  The ingredients and instructions are stored as arrays.  This class has a class method to scrape the name and URL of each recipe in a Category, as well as an instance method to scrape the ingredients of and instructions for a single recipe.

I was torn as to where to place the class method to scrape all recipes from a single category.  I decided to place this functionality in KetoRecipe because the application does not need to know all recipes in all categories.  It only needs a list of recipes in a category once the user selects a category.  The benefit of only scraping a list of recipes in single a category are (1) there is less data to download, improving performance of the application, and (2) the application doesn't consume more of Keto Connect's bandwidth than necessary.

The last class is the Menu class.  Originally, the menu was in the Category class.  However, when I was writing the KetoRecipe class, I noticed that I would simply be copying and pasting the menu functionality.  Both the category menu and the recipe menu essentially display a numbered list and prompt the user to enter a number.  I decided to follow the DRY principle (don't repeat yourself), and created a generic menu class that could be used by both Category and KetoRecipe.

This project was interesting because I was able to practice two skills.  First, I gained more experience with Nokogiri, a web scraping tool.  Second, I designed and built a project from scratch.  While I've been coding off-and-on for some time in various languages, I haven't really created much using my skills.  This project was a great opportunity to practice thinking about how to design an application from beginning to end.  Also, I was able to use the Ruby skills that I've acquired through the Flatiron course.
